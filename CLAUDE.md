## Project Overview

SIMRS (Sistem Informasi Manajemen Rumah Sakit) — an Electronic Medical Record System built with TanStack Start (full-stack React SSR).

## Commands

```bash
bun install                    # Install dependencies
bun --bun run dev              # Dev server on port 3000
bun --bun run build            # Production build
bun --bun run preview          # Preview production build
bun --bun run test             # Run tests (vitest)
bun --bun run lint             # ESLint
```

## Architecture

### Stack

TanStack Start + TanStack Router (file-based) + Vite 8 + Nitro server runtime + React 19 + TypeScript 6

- **Styling**: Tailwind CSS 4 + shadcn/ui (radix-nova style, components in `src/components/ui/`)
- **Database**: PostgreSQL via Drizzle ORM (`src/db/schema.ts`, `src/db/relations.ts`)
- **Auth**: better-auth with Redis secondary storage, admin + phone number plugins (`src/lib/auth.ts`)
- **API**: oRPC with TanStack Query integration, Redis caching (60s TTL), retry on 429/503 (`src/orpc/`)
- **Cache/Sessions**: Redis via ioredis (`src/lib/redis.ts`)
- **Validation**: Zod 4
- **Env**: Validated via t3-oss/env-core (`src/env.ts`)

### Path Aliases

`@/*` and `#/*` both resolve to `./src/*` (configured in tsconfig.json).

### Routing (TanStack Router file-based)

```
src/routes/
  __root.tsx                   # Root layout (providers, devtools)
  _dashboard/
    route.tsx                  # Dashboard layout (sidebar + header)
    index.tsx                  # Dashboard page
  api/
    rpc.$.ts                   # oRPC handler (all RPC calls go through here)
    health.ts                  # GET /api/health
    files.$.ts                 # Static file serving from storages/public/
    assets.$.ts                # Static file serving from storages/public/
  auth/
    route.tsx                  # Auth layout wrapper
    sign-in.tsx                # Sign-in page
```

### oRPC Pattern

- Router definitions: `src/orpc/router/` (add new procedures here)
- Root router: `src/orpc/router/index.ts` (compose sub-routers)
- Client: `src/orpc/client.ts` (isomorphic — server uses direct invocation, client uses HTTP to `/api/rpc`)
- Middleware: `src/orpc/middleware/` (auth, rate-limiter)
- Base with caching: `src/orpc/index.ts`
- API handler: `src/routes/api/rpc.$.ts`

Client-side data fetching uses `@orpc/tanstack-query` hooks generated from the router type.

### Database Schema

Four tables in `src/db/schema.ts`: `user`, `session`, `account`, `verification` — all use uuidv7 primary keys. Drizzle client is in `src/db/index.ts`. Migrations output to `./drizzle/`.

### Auth Flow

- better-auth handles email/password, session management, admin features
- Redis used as secondary session storage
- Legacy auth (`legacySignIn`/`legacyRemoveSession` in `src/orpc/router/auth.ts`) proxies to external `VITE_API_URL` for backward compatibility
